Hexo + Github Pages 搭建个人独立博客

发表于 2017-03-28

这才是明确意义上我的第一篇技术blog吧~前端纯小白，目前研0，之后3年准备奋力向前端技术努力！搭建的整个过程是一天多，当然只是一些零碎的时间，记录下来希望给大家一些帮助~有问题欢迎留言啊！
Tips：本文原理很少，因为博主小白不懂原理，但一定是尽量给了大家详细的步骤和大家可能出现的错误的解答~
本文涉及很多的安装和下载，如果发现速度很慢，请大家注意及时翻墙呢~

安装Node.js

下载Node.js在这里大家选择自己所需要的版本下载就好了
安装Node.js

安装Hexo

大家打开win cmd的命令行也是一样可以的，这里贴出的代码是linux系统下面的

接下来介绍几个常用的Hexo下面的命令：

  • 1.hexo generate(hexo g) 生成静态文件，会在当前目录下生成一个新的叫做public的文件夹（其实我也不太懂，先跟着用吧）
  • 2.hexo server（hexo s）启动本地web服务，用于博客的预览
  • 3.hexo deloy（hexo d）部署博客到远端（比如github，heroku等平台）
  • 4.hexo new “postName” （新建标题叫postName的文章）
  • 5.hexo new page “pageName” (新建标题叫pageName的页面）
  • 6.hexo clean 就是经常要clean一下，不管什么开始之前都要clean一下，好像就是clean掉缓存

安装主题

目前我安装的是Next主题，大家如果和我一样是小白就先跟着教程安装这个把，等到之后会了就可以尝试更换别的主题了~~

在这里提醒大家注意一点，下载主题的时候还是选择稳定版本吧，我一开始就是用的克隆，可能就是版本问题折腾了我好久，当然也可能是能力问题。。。。然后在修改配置文件时希望大家一定注意属性后面的空格，总之空格很重要啊！！一定要注意！

调整好配置文件之后，请顺序执行这四个命令

  • 1.hexo clean
  • 2.hexo g
  • 3.hexo d
  • 4.hexo server

正常情况下是不会有什么问题的啊~~大家可以看看自己本地已经部署好的页面是否满意

Github Page设置

关于github page 的设置网上的教程很多了，无非以下几个步骤

  • 1.注册账户
  • 2.新建仓库（仓库的名称和自己账号的名称保持一致）

特别提醒，个人主页的网站内容是在master分支下的，别的话，这里没有链接别的blog了，大家可以查查看，很简单啦~

部署Hexo到Github Page

这一步是最关键的一步了，也就是我们如何把本地的page弄到我们自己在github的仓库中去。
首先我们提前安装扩展：

  • $ npm install hexo-deployer-git –sava

没有设置SSH key的请在本机生成SSH key并进行设置链接给大家的是官方的帮助文件，纯英文大家使用以下chrome自带的翻译页面就好，我就是这样用的，相当方便，也不存在什么错误，就是及时把仓库修改成自己的仓库名称就好。同时值得注意的是，命令行的命令请务必看一下它文件所属的地址，这样你才知道你的命令行是应该在哪里打开，一定注意的就是地址，别的就没什么了~

然后我们要在系统配置文件_config.xml中进行修改

请记得把repo改成自己的啊~

然后在命令行执行

  • hexo d
    即可完成部署
